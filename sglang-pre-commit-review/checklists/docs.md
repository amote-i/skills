# 文档检视检查清单

适用于 `docs_new/` 及所有 markdown/mdx/rst/ipynb 变更。

> **作用域限制：** 本清单仅覆盖 `docs_new/` 目录。`docs/`（旧版）、`examples/`、顶层 `README.md` 不在检视范围内。
> **最后验证：** 2026-05

---

## 1. 文档站点（`docs_new/`）

### 结构与位置
- [ ] 新文档放在 `docs_new/docs/`（**不是**旧版 `docs/`）
- [ ] 文档内容使用 `.mdx` 扩展名（当前绝大多数文件使用 `.mdx`）。仅项目元文件（`README.md`、`AGENTS.md`、`CONTRIBUTING.md`）和遗留内容使用 `.md`。另有少量 `.rst` 遗留文件和 `.ipynb` 教程。
- [ ] 遵循现有目录结构和命名规范
- [ ] 新增顶层页面已在 `docs_new/docs.json` 导航中引用

### 内容质量
- [ ] 准确：代码示例可实际运行并产生所述输出
- [ ] 完整：覆盖特性的完整流程（安装、使用、配置、排障）
- [ ] 时效：引用当前 API，非已弃用的签名
- [ ] 链接有效：内部链接可解析，外部链接非 404
- [ ] 版本感知：标注特性新增或变更的版本（如为近期）

### 文档中的代码示例
- [ ] 示例自包含（复制即可运行）
- [ ] 使用真实模型名和参数（非 `YOUR_MODEL` 等占位值）
- [ ] 在有帮助处展示预期输出
- [ ] 包含用户常犯错误的错误处理代码

## 2. 旧版文档（`docs/`）

> **作用域限制：** `docs/` 不在检视范围内。若 diff 中意外出现 `docs/` 的变更，应标记为"作用域外"并跳过。以下规则仅在用户明确要求检视 `docs/` 时参考。

- [ ] **不得修改 `docs/`** — pre-commit hook `check-no-docs-changes` 会拦截
- [ ] 如确需修改旧文档，必须同步将内容迁移到 `docs_new/`

## 3. API 文档（Docstring）

> **注意：** 当前代码库 docstring 覆盖率较低（公共函数 ~22%，公共类 ~32%）。以下为鼓励方向，仅对 `entrypoints/` 公共 API 层和 `server_args.py` 的 help text 建议作为重要，其余均为次要。

### 覆盖范围
- [ ] 新公共函数/方法有 docstring（次要，`entrypoints/` 除外）
- [ ] 新类有类级 docstring 说明用途和用法（次要）
- [ ] 新 CLI 参数在 `server_args.py` 中有 help text（重要）
- [ ] 新配置项在相关配置文档中有记录（次要）

### Docstring 质量
- [ ] Args 部分：每个参数有类型和描述
- [ ] Returns 部分：返回类型和含义有文档
- [ ] Raises 部分：异常有文档（尤其公共 API）
- [ ] docstring 中的类型与实际类型注解一致
- [ ] docstring 风格与同文件其他代码一致（Google 风格或 NumPy 风格）

### 无需文档的情况
- [ ] 私有方法（`_前缀`）：docstring 可选
- [ ] 显而易见的 getter/setter：无需 docstring
- [ ] 行为未变的重写方法：简要注释即可

## 4. 交叉引用检查

### 代码变更时检查文档
- [ ] 新 CLI 标志 → CLI 参考和相关特性文档中有记录
- [ ] 新 server 参数 → `server_args.py` 的 help text **及**文档中均有
- [ ] 新 API 端点 → 如相关，OpenAI 兼容性文档已更新
- [ ] 新模型支持 → `docs_new/` 中支持模型列表已更新
- [ ] 破坏性 API 变更 → 迁移指南或变更日志条目
- [ ] 弃用特性 → 已添加弃用通知和时间线

### 文档变更时对照代码验证
- [ ] 引用的函数/类实际存在且签名匹配描述
- [ ] 文档中的代码片段与当前 API 匹配（参数名、返回类型）
- [ ] 文档中的默认值与代码实际默认值一致
- [ ] 文档中提及的特性标志在 `server_args.py` 中存在

## 5. 风格与格式

- [ ] Markdown 格式正确（标题层次、表格完整、代码块闭合）
- [ ] 标题层级一致（不跳级：H1 → H3）
- [ ] 可用 Markdown 时不使用原始 HTML
- [ ] 图片有 alt text
- [ ] 术语与文档其余部分一致
- [ ] 除有意本地化的内容外，无非预期中文字符

## 6. 特殊文档类型

### 架构/设计文档
- [ ] 图表是最新的且与实现一致
- [ ] 组件关系描述准确
- [ ] 权衡和设计决策有解释

### 基准测试/性能文档
- [ ] 硬件规格已文档化（GPU 型号、数量、内存）
- [ ] 软件版本已指定（SGLang 版本、CUDA 版本、PyTorch 版本）
- [ ] 基准测试方法有描述（预热、测量时长、指标定义）
- [ ] 相同硬件和配置下结果可复现

### 贡献指南
- [ ] 步骤顺序正确
- [ ] 命令可直接复制运行
- [ ] 前置条件已列出（Python 版本、CUDA 版本等）

## 严重等级映射指南

| 分类 | 严重 | 重要 | 次要 |
|------|------|------|------|
| 位置错误 | 修改了旧版 `docs/` | 新页面未加入 `docs_new/docs.json` 导航 | 文件扩展名不一致 |
| 准确性 | 代码示例无法运行、API 签名错误 | 默认值过时、缺少 import | 描述略有误导 |
| 覆盖范围 | 新 CLI 参数无 help text | 新 API 端点无文档记录 | 新公共函数缺少 docstring、缺少 `Raises` 部分 |
| 交叉引用 | 文档引用了不存在的函数 | 代码默认值与文档默认值不一致 | 小术语不一致 |
| 风格 | Markdown 表格损坏、代码块未闭合 | 跳过标题层级 | 小格式问题 |
